# 儿童启蒙系统 - KID Happy Learning

## 1. 目标用户

- 3~5岁学龄前儿童，家长陪同或独立使用。
- 设备：iPad / 安卓平板 / 电视（16:9，触控智慧屏优先）/ 手机。
- **交互方式**：触控为主；电视端若检测到无触控（`navigator.maxTouchPoints === 0`），自动降级为方向键+Enter键盘导航。

---

## 2. 核心学习模块

| 模块 | 内容 | 互动形式 | 启蒙目标 |
|------|------|----------|----------|
| 🔤 字母乐园 | A~G（可扩展）大写字母，每个字母配一个代表单词+emoji | 点击卡片 → 播放预录发音（字母名+单词）<br>同时展示单词示例 + 故事文字弹窗 + 语音讲述 | 认识字母形态，建立字母与词汇的初步联系 |
| 🍎 单词小屋 | 常见动物/水果/自然（cat, dog, sun, apple, bird, fish） | 点击卡片 → 播放预录英文发音，显示可爱emoji | 积累基础听力词汇，语音模仿 |
| 🇨🇳 汉字启蒙 | 简单象形字：人、口、日、月、水、火 | 点击卡片 → 播放预录中文发音 + 组词（如"人口"） | 培养汉字兴趣，感知字形与意义 |

### 2.1 故事功能说明
- 每个字母/单词配一段 **1-2句话、20-40字** 的短故事
- 展示形式：文字弹窗 + 预录语音讲述（由家长/老师提前录制）
- 语言：与卡片语言一致（中文卡片中文故事，英文卡片英文故事）
- **暂不实现自动生成**，保持内容质量和离线可用性

## 3. 趣味互动机制

### 3.1 ✨ 星星奖励系统
- 每完成一次"魔法挑战"游戏，答对即可获得星星
- 星星用 **localStorage** 持久化存储（30天自动过期）
- 累计星星给予鼓励动画（CSS animation，弹性缩放+旋转）
- 星星展示：首页顶部显示当前星星总数

### 3.2 🎮 魔法挑战（综合测验）
- 随机从三个模块中抽取知识点（字母/单词/汉字），显示问题和三个选项（正确 + 两个随机干扰项）
- **出题逻辑**：在全部19题（7字母+6单词+6汉字）完成前不重复；循环后可重复
- **难度梯度**：连续答对3次，该题后续出现频率降低；连续答错2次，增加练习频率（未来自适应功能）
- 答对 → 加1星，播放欢快音效 + 鼓励语音（"Good!" / "你真棒！" / "太厉害了！"）
- 答错 → 温和音效 + "再试一次哦~" 语音 + 高亮展示正确答案卡片，不扣星

### 3.3 🔊 点读功能
- 所有卡片均可点击发音，强化听觉记忆
- 使用 **Howler.js** 管理音频播放（预加载、并发、回退HTML5 Audio）
- 音频格式：**MP3**（主） + **WebM**（备用），覆盖所有浏览器

### 3.4 🌸 视觉与交互规范
- **触控目标**：最小 **80×80px**，间距 ≥ 12px（行业标准 75×75px+）
- **色彩**：明亮主色调（红/蓝/黄/绿），高对比度，避免视觉疲劳
- **动画**：弹性按压动画（active状态缩放0.95），300ms缓动
- **字体**：18-19px，儿童友好字体（圆体/Comic Sans类）
- **布局**：扁平化单屏Hub，三个模块并列展示，无嵌套导航

## 5. 技术方案

| 技术 | 方案 | 理由 |
|------|------|------|
| **音频播放** | Howler.js（MP3/WebM，预录音频） | 100%离线可用，音质稳定，行业主流做法（Duolingo ABC、Khan Academy Kids均用预录） |
| **离线运行** | Service Worker（Cache API）缓存HTML/CSS/JS/音频 | 满足"可离线运行"需求 |
| **数据存储** | localStorage（星星、进度），30天过期 | 简单可靠，适合儿童App短期记忆特点 |
| **响应式布局** | CSS Grid + Flexbox + media queries | 完美适配16:9电视、iPad、安卓平板、手机 |
| **多语言发音** | 预录音频文件（中文/英文分开录制） | 不依赖Web Speech API，解决离线矛盾 |
| **交互检测** | `navigator.maxTouchPoints` 检测触控能力 | 自动切换触控/键盘导航模式 |

### 5.1 为什么不用Web Speech API？
- **矛盾**：需求要求"可离线运行"，但Speech API需要网络
- **可靠性**：儿童语音识别率低，发音质量参差不齐
- **解决方案**：预录音频 + Howler.js，参考Duolingo ABC/Khan Kids行业标准

## 6. 不明确需求与最终决策

| # | 不明确点 | 最终决策 | 理由 |
|---|---------|---------|------|
| Q1 | 电视端交互方式 | **触控智慧屏优先** + 无触控时降级为键盘导航 | 3-5岁儿童无法使用遥控器精准操作；触控是行业标准 |
| Q2 | 发音方案 | **预录音频（MP3）+ Howler.js** | 完全满足离线需求，音质稳定，行业主流 |
| Q3 | 故事功能范围 | **预录语音讲述 + 文字弹窗**（暂不自动生成） | 保证内容质量和离线可用性，3-5岁需成人讲述 |
| Q4 | 星星持久化 | **localStorage，30天过期** | 符合儿童App短期记忆特点，家长可清除 |
| Q5 | 家长功能 | **最小实现**：长按Logo 3秒进入家长区（查看进度+清除数据） | MVP阶段最小实现，未来可扩展完整家长端 |

## 7. 行业最佳实践参考

### 7.1 设计原则（来源：Smashing Magazine, CAS.ai, Antikode）
- **扁平化导航**：所有主要功能一个屏幕展示，避免嵌套（"house → rooms → games"会让儿童迷失）
- **即时反馈**：每次操作300ms内给出视觉/听觉反馈，建立因果感
- **时间分块**：每次活动2-5分钟，之后可显示"休息一下"页面
- **内在动机**：星星奖励作为辅助，重点是学习本身的乐趣（故事、动画、探索感）

### 7.2 触控与视觉规范
- 触控目标：最小80×80px（Smashing Magazine建议75×75px+）
- 字体大小：18-19px，儿童友好圆体
- 色彩：明亮主色（红/蓝/黄/绿），高对比度
- 间距：按钮间距≥12px，防止误触

### 7.3 参考产品
| 产品 | 特色 | 可借鉴点 |
|------|------|----------|
| **Duolingo ABC** | 离线运行，预录音频，游戏化学习 | 音频方案、离线架构 |
| **Khan Academy Kids** | 完全免费，无广告，家长区，进度追踪 | 家长功能、隐私保护 |
| **SplashLearn** | 个性化学习路径，短测验onboarding | 自适应学习（未来） |
| **CAS.ai研究** | 扁平导航，避免抽象货币，进度条可视化 | UI/UX设计原则 |

## 11. 隐私与数据治理（COPPA合规）

### 11.1 数据收集原则
- **不收集**任何个人身份信息（PII）：无姓名、生日、照片上传
- **仅本地存储**：星星数、学习进度存localStorage，不上传服务器
- **无第三方追踪**：无广告SDK，无分析追踪（参考Khan Kids零广告原则）

### 11.2 隐私声明
- 首次打开显示隐私声明弹窗（可滚动阅读，点击"我知道了"关闭）
- 声明内容：仅本地存储学习进度，无数据上传，家长可随时清除数据

### 11.3 数据过期
- localStorage数据30天自动过期（通过版本号检测清除旧数据）
- 家长区提供"清除所有数据"按钮

## 12. 无障碍设计（WCAG 2.1 AA for Children）

| 项目 | 要求 | 实现方式 |
|------|------|----------|
| 对比度 | 文字与背景≥4.5:1 | 颜色选择工具验证，避免浅灰背景+白字 |
| 字体大小 | 最小18px | CSS全局设置，触控卡片内文字≥18px |
| 键盘导航 | 支持Tab/方向键+Enter | 所有交互元素可focus，显示focus环 |
| 屏幕阅读器 | 图标配文字标签 | `aria-label` 或可视文字，如"字母A卡片" |
| 高对比度模式 | 家长可切换 | 添加"高对比度"开关（家长区），切换CSS变量 |
