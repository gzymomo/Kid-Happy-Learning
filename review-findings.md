## 🔴 Critical Issues

### 1. Magic Challenge (Quiz) has no UI entry point
**Files**: `index.html`, `js/app.js:332-492`  
**Requirements**: §3.2 (Magic Challenge) with acceptance criteria Q-01 through Q-04  
**Issue**: The quiz modal and all quiz logic (`initQuiz()`, `showQuiz()`, `pickQuestion()`, `handleQuizAnswer()`) are fully implemented, but **no button or trigger exists in the UI to open the quiz**. The home view only offers 3 category cards; the learning view only shows content cards. The entire "魔法挑战" feature is inaccessible to users.  
**Fix**: Add a fourth card/button on the home view (e.g., a "🎮 魔法挑战" card) wired to call `showQuiz()`.

### 2. Service Worker does not cache audio files — offline audio breaks
**Files**: `sw.js:3-9`  
**Requirements**: §5 (offline operation), W-03 (offline audio), G-01 (offline core functionality)  
**Issue**: The SW precache list only includes HTML, CSS, JS, JSON, and Howler.js CDN — **no audio files** (`audio/letters/*.mp3`, `audio/words/*.mp3`, `audio/chinese/*.mp3`, `audio/stories/*.mp3`, `audio/effects/*.mp3`). The fetch handler returns a `204 No Content` empty response when audio is requested offline. Audio playback is completely broken offline.  
**Fix**: Either (a) precache audio files, (b) implement runtime caching with a cache-first strategy for the `/audio/` path, or (c) lazy-cache audio on first play.

## 🟡 Medium Issues

### 3. Encouragement phrases use Web Speech API (breaks offline)
**File**: `js/app.js:465`  
**Requirements**: §3.2 (答对→播放欢快音效+鼓励语音), §5 (offline reliability)  
**Issue**: In `handleQuizAnswer()`, encouragement phrases (`"Good job!"`, `"你真棒！"`, `"太厉害了！"`) are spoken via `speak()` (Web Speech API TTS). This requires network connectivity and will fail offline. The `playAudioOrSpeak()` function exists but is not used here. The `rewards.encouragePhrases` array exists in content.json but no corresponding audio file paths exist.  
**Fix**: Add audio file paths for each encouragement phrase in `content.json` under `rewards` and use `playAudioOrSpeak()` instead of `speak()`.

### 4. Parent zone long-press logo entry not implemented
**File**: `js/app.js:545-554`  
**Requirements**: §14.1 (长按Logo 3秒 → 进入家长区; or 连续点击右下角空白处5次)  
**Issue**: The only implemented entry method is the 5-click footer dot trigger (`initFooterTriggers()`). The **long-press (hold 3 seconds) on the logo** is not implemented. There's no identifiable logo element in `index.html` — only a version text in the footer.  
**Fix**: Add a logo element (e.g., the app title or an image) with a `longpress`/`mousedown`+`setTimeout` handler, or document that only the 5-click method is used and update requirements.

### 5. `visitedLetters/visitedWords/visitedChinese` arrays accumulate duplicates
**File**: `js/app.js:228, 249, 270`  
**Requirements**: §14.2 (parent zone stats) — accuracy of progress tracking  
**Issue**: Every click on a card pushes the item ID to the visited array **without deduplication**. If a child clicks the same card 10 times, the array grows by 10 entries. Parent zone stats (`state.visitedLetters.length + '/7'`) report inflated numbers.  
**Fix**: Check for existing ID before pushing, e.g.:
```js
if (state.visitedLetters.indexOf(letter.id) === -1) {
  state.visitedLetters.push(letter.id);
}
```

### 6. Hardcoded stat denominators break extensibility
**File**: `js/app.js:518-520`  
**Requirements**: §4.2 (JSON-driven extensibility), L-03 (add letter H without code changes)  
**Issue**: Parent zone stat denominators are hardcoded strings (`'/7'`, `'/6'`, `'/6'`) instead of being computed from `content.letters.length`, `content.words.length`, `content.chinese.length`. Adding a new letter/word/character to `content.json` will not update the displayed max counts without also editing `app.js`.  
**Fix**: Compute denominators dynamically:
```js
document.getElementById('statLetters').textContent = state.visitedLetters.length + '/' + content.letters.length;
```

## 🔵 Minor / Enhancement

### 7. Story modal does not auto-play audio
**File**: `js/app.js:302-310`  
**Requirements**: §2.1 (预录语音讲述 — pre-recorded audio narration)  
**Issue**: When a letter card is clicked, the letter name plays, then after 1s the story modal appears — but the story audio does not auto-play. The user must click "听故事" button. The requirements imply the story narration should play automatically when the modal opens.  
**Note**: This may be intentional to avoid autoplay restrictions; consider documenting the design decision.

### 8. No today's study duration in parent zone
**File**: `js/app.js:516-526`  
**Requirements**: §14.2 (今日学习时长 — today's study duration)  
**Issue**: The parent zone shows stars total and per-module completion counts, but the "今日学习时长" (today's study time) stat is not implemented. This would require tracking session start times or cumulative active time.

### 9. `manifest.json` not cached by Service Worker
**File**: `sw.js:3-9`  
**Issue**: The PWA manifest is not in the SW precache list. If the user is offline and reinstalls, the manifest may not be available. Minor impact because manifests are only used on install.

### 10. Audio `preload: false` causes first-play delay
**File**: `js/app.js:122`  
**Issue**: `AudioManager.loadSound()` sets `preload: false`. On first click, Howler.js must fetch and decode the audio file, which may exceed the 300ms acceptance criterion (L-01) for the first play of each card. Subsequent plays are fine because Howler caches decoded audio.
